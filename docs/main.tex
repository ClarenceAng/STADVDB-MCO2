% TODO: Write 150-250 words summarizing:

% - Purpose and method (three-node distributed database system)

This technical report presents the design, implementation, and evaluation of a three-node distributed database system that supports concurrent multi-user access and transaction management. The system implements data fragmentation and replication strategies across the nodes, with Node 1 serving as the central node containing all data, while Nodes 2 and 3 contain horizontal partitions based on specified fragmentation criteria. We developed and tested concurrency control mechanisms to maintain data consistency across concurrent read and write operations, evaluating performance across different isolation levels (read uncommitted, read committed, repeatable read, and serializable). Additionally, we implemented global failure recovery strategies to handle node failures and transaction failures during replication. Our experimental results demonstrate the effectiveness of our update and replication strategies in maintaining database consistency while supporting high transaction throughput. The findings provide insights into the trade-offs between isolation levels, concurrency, and consistency in distributed database systems.
\end{abstract}
